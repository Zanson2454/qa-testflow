# E2E 验收标准（首版）

> 说明：该标准用于 `PRD -> 测试用例 -> E2E验收` 的统一判定，保证 AI 输出结果可审计、可复核。

## 1. 验收对象

- 输入：已确认版本的 PRD 与对应用例映射文档。
- 执行：基于 Playwright 的浏览器端到端测试。
- 输出：通过/失败结论 + 证据包 + 风险结论。

## 2. 通过标准（pass）

同时满足以下条件才可判定通过：

1. P0 用例全部通过。
2. 每条通过用例均有可复核证据（至少包含 trace 或截图）。
3. 关键业务链路满足“双层信号”：
  - 页面层：页面显示行为符合预期；
  - 数据层：关键业务数据加载/拦截信号符合预期。
4. 无阻断级（blocker）未解决问题。

## 3. 失败标准（fail）

存在任一情况即判定失败：

- 任一 P0 用例失败。
- 证据缺失，无法复核执行结论。
- 页面层通过但数据层信号不符合预期（例如页面打开但核心数据不可用）。
- 出现高风险安全或权限异常（例如未公开页面可直接访问）。

## 4. 证据标准

每条失败用例必须至少包含：

- 失败步骤截图（必要）
- trace 文件（必要）
- 关键日志摘要（建议）
- 失败结论与影响说明（必要）

每轮验收必须输出统一报告，包含：

- 执行时间与环境
- 用例通过率（P0/P1）
- 失败清单与风险分级
- 结论：`passed: true|false`

## 5. PM 验收闸门

- 技术通过不等于业务最终通过。
- AI 负责“测试执行与证据输出”，PM 负责“业务预期确认”。
- PM 闸门结果字段建议：
  - `pm_gate`: pass | fail | clarification_needed
  - `pm_comment`: 验收说明

## 6. 版本治理

- PRD 版本升级后，必须同步更新用例映射版本。
- 用例版本变更必须记录 `change_reason`。
- 验收报告必须标注 `prd_version` 与 `case_set_version`。
